# 论文阅读笔记 Skill

## 何时使用
- 阅读 PDF 论文
- 分析研究方法和实验设计
- 提取论文关键信息（方法、实验、结论）
- 从 Zotero 获取文献并整理笔记
- **深度分析论文中的图表**（使用 MinerU 提取）
- 批判性分析研究质量

---

## 🆕 MinerU 图表提取工作流（推荐）

> **核心价值**：论文中的 Figure/Table 包含大量信息，但传统 OCR 无法理解图表含义。
> 本工作流通过 MinerU 提取高质量图表，再用 Vision LLM 深度分析。

### MinerU 安装与使用

**安装（首次使用）**：
```bash
# 推荐使用 uv（更快）
uv pip install magic-pdf

# 或使用 pip
pip install magic-pdf

# 首次运行会自动下载模型（约 2GB）
```

**基础用法**：
```bash
# 解析单个 PDF
magic-pdf -p /path/to/paper.pdf -o /path/to/output/ -m auto

# 参数说明:
# -p: PDF 文件路径
# -o: 输出目录
# -m: 模式（auto/txt/ocr），推荐 auto
```

**输出文件说明**：
| 文件 | 内容 | 用途 |
|------|------|------|
| `paper.md` | 完整 Markdown | 主体文本阅读 |
| `images/*.png` | 所有图表 | Claude Vision 分析 |
| `content.json` | 结构化数据 | 程序化处理 |

### 使用示例

**用户输入**：
```
帮我用 MinerU 工作流读一下这篇论文：OpenVLA
```

**Claude 执行流程**：
1. 从 Zotero 获取 OpenVLA 论文 PDF
2. 运行 MinerU 解析：`magic-pdf -p openvla.pdf -o ./openvla_output/ -m auto`
3. 读取生成的 Markdown 和 images/
4. 对每张图片执行 Vision 分析
5. 按 Section 1-4 模板整合笔记，将图表分析嵌入对应位置
6. 询问是否导出到飞书

### 何时使用 MinerU 工作流

| 场景 | 推荐方式 | 原因 |
|------|----------|------|
| 论文图表很多（>5张） | ✅ MinerU 工作流 | 批量提取 + 深度分析 |
| 需要分析复杂架构图 | ✅ MinerU 工作流 | Vision LLM 理解图表 |
| 论文含大量实验表格 | ✅ MinerU 工作流 | 保留表格结构 |
| 快速浏览/短论文 | ⚪ 传统方式 | Zotero 全文即可 |
| 只需要读摘要/结论 | ⚪ 传统方式 | 无需图表分析 |

## 推荐分批输出策略

由于LLM的输出token限制，**论文越长，一次性输出所有Section质量会越低**。建议分批完成：

| 批次 | 内容 | 预计时间 |
|-----|------|---------|
| **第一批** | Section 1 + Section 2（概览与背景） | ~35分钟 |
| **第二批** | Section 3: Method（单独完成） | ~40分钟 |
| **第三批** | Section 4: Experiments（单独完成） | ~30分钟 |
| **第四批** | Section 5: Conclusion（可选） | ~5分钟 |

### 为什么要分批？

| 一次性输出全部 | 分批输出 |
|--------------|---------|
| ❌ 后面的Section质量下降 | ✅ 每个Section都保持高质量 |
| ❌ 容易触达token上限被截断 | ✅ 避免输出截断 |
| ❌ Section 3/4可能过于简略 | ✅ Method和Experiments得到充分展开 |
| ⚠️ 适合短论文（<10页） | ✅ 适合所有长度的论文 |

### 具体操作

**对话1：**
```
请帮我阅读这篇论文，先完成Section 1和Section 2的笔记。
[上传论文PDF 或 从Zotero获取]
```

**对话2：**
```
现在请帮我完成Section 3: Method部分的笔记。
重点关注公式解析和Pipeline流程。
```

**对话3：**
```
最后请帮我完成Section 4: Experiments部分的笔记。
重点整理关键发现和消融实验。
```

### Section 1 填写指南

**填写顺序**：
1. 快速浏览Abstract、Introduction、Conclusion
2. 填写1.1：重点关注Introduction中的motivation部分

3. 填写1.2：重点关注方法概述和贡献声明
4. 填写1.3：通常在Introduction末尾或Abstract中

**关键技巧**：
- ✅ 用论文中的图表编号标注证据（如"见图2"）
- ✅ 案例分析尽量用论文原文的例子，不要自己编
- ✅ 如果有图片，务必结合图片和caption分析
- ✅ 数字要准确，最好直接从论文的表格/Abstract中摘录

- [ ] 1.1至少给出了1个具体场景案例

**公式解析技巧（核心）**：

**示例：如何解析一个公式**

**公式：动态词表构建**

$$
E_{dyn} = [E_{text}; P_{ref}], \quad P_{ref} = f_{vp}(F_{patch}) \in \mathbb{R}^{N' \times d}
$$

- **输入**：
  - $E_{text} \in \mathbb{R}^{V_{text} \times d}$：原始文本embedding表
  - $F_{patch} \in \mathbb{R}^{N' \times d}$：经过projector处理的patch features
- **计算过程**：
  1. $f_{vp}$（包含LayerNorm和低秩线性投影）将 $F_{patch}$ 投影为视觉参考原型 $P_{ref}$
  2. 将 $P_{ref}$ 拼接到 $E_{text}$ 后面，形成动态扩展的词表
- **输出**：$E_{dyn} \in \mathbb{R}^{(V_{text}+N') \times d}$，包含文本和视觉token的统一embedding表
- **直觉理解**：每次forward pass时，将当前图像的patch features动态加入词表，使得LLM可以"引用"图像中的特定区域。

**常用LaTeX符号**：
- 向量/矩阵：$\mathbf{W}$, $\mathbf{h}_t$
- 维度：$\mathbb{R}^{N \times d}$
- 下标：$F_{patch}$, $h_t$
- 希腊字母：$\alpha$, $\beta$, $\gamma$
- 求和：$\sum_{i=1}^{N}$
- 范数：$\|\cdot\|_1$, $\|\cdot\|_2$
- 损失函数：$\mathcal{L}_{CE}$
